# Smart Waiting Area Triage for Pharmacy and Outpatient Screening

## 1. ชื่อโจทย์
**Smart Waiting Area Triage for Pharmacy and Outpatient Screening**

## 2. สถานการณ์
ที่หน้าเวชระเบียนหรือจุดคัดกรองก่อนรับยา/พบแพทย์ ผู้ป่วยบางรายมีอาการหน้ามืด อ่อนเพลีย หรือความดันต่ำระหว่างรอ หากเจ้าหน้าที่ไม่สังเกตเห็นทันที อาจทำให้ผู้ป่วยมีอาการทรุดลงได้

## 3. เป้าหมาย
สร้างระบบติดตามสัญญาณชีพเบื้องต้นแบบ real-time เพื่อ:

- เฝ้าระวังผู้ป่วยเสี่ยง
- ลดการตกหล่นของผู้ป่วยที่มีอาการแย่ลงระหว่างรอ
- ช่วยเจ้าหน้าที่ตัดสินใจเรียกพบแพทย์ได้รวดเร็วขึ้น
- แสดงผลบน dashboard ให้เข้าใจง่าย

## 4. ลำดับการทำงาน

1. ผู้ป่วยมาถึงจุดคัดกรอง
2. เจ้าหน้าที่วัด BP, pulse และ temperature
3. ค่าที่วัดได้ถูกส่งเข้า Raspberry Pi / Sense HAT
4. Sense HAT publish ข้อมูลผ่าน MQTT ไปยัง Node-RED
5. Node-RED ประเมินระดับความเสี่ยงของผู้ป่วย
6. Node-RED ส่งข้อมูลต่อไปยัง ThingsBoard
7. Dashboard แสดงสถานะด้วยสี เช่น สีเหลืองหรือสีแดง
8. หากอยู่ในระดับ critical ระบบจะแจ้งเตือนเจ้าหน้าที่ทันที
9. ผู้ป่วยถูกพาเข้าพบแพทย์ก่อนคิว

## 5. ตัวอย่าง Alert Message

Node-RED อาจส่งข้อความแจ้งเตือนไปยัง LINE หรือแสดงบน dashboard ดังนี้:
